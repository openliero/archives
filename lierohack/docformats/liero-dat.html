Liero data (DAT) file format

Offset Field Size Contents
0000h Max bonuses 1 Byte This is the maximum number of bonuses in the game.
0001h Loading times 1 Word This is the loading times setting. The number is the desired percentage of the default loading times. Treated as a Word in the game, but as a Short in the menu.
0003h Number of lives 1 Short When the game mode is Kill'em all, this number is the lives setting.
0005h Time to lose 1 Short When the game mode is Game of Tag, this number is the Time to Lose in seconds.
0007h Flags to win 1 Short When the game mode is Capture the Flag or Simple CtF, this number is the Flags to win setting.
0009h Screen sync 1 Byte Screen sync setting. Possible values:
0 = OFF
1 = ON
000Ah Map 1 Byte Minimap setting. Possible values:
0 = OFF
1 = ON
000Bh Controller player 1 1 Byte Controller setting for player 1. Possible values:
0 = Human
1 = CPU
000Ch Controller player 2 1 Byte Controller setting for player 2. Possible values:
0 = Human
1 = CPU
000Dh Random level 1 Byte Random level setting. Possible values:
0 = Custom level
1 = Random level
000Eh Amount of blood 1 Word Amount of blood in the game. The number is the desired percentage of the default amount of blood. Treated as a Word in the game, but as a Short in the menu.
0010h Game mode 1 Byte Game mode setting. Possible values:
0 = Kill'em All
1 = Game of Tag
2 = Capture the Flag
3 = Simple CtF
0011h Names on bonuses 1 Byte Names on bonuses setting. Possible values:
0 = OFF
1 = ON
0012h Reload level 1 Byte Reload level setting. Determines if the level has to be reloaded for every new game. Possible values:
0 = OFF
1 = ON
0013h Shadows 1 Byte Shadows setting. Possible values:
0 = OFF
1 = ON
0014h Weapon options 40 * 1 Byte Weapon avaliability for each of the 40 weapons (sorted by internal weapon number, see useful lists). Possible values:
0 = Menu
1 = Bonus
2 = Banned
003Ch Color player 1 3 * 1 Byte Color settings for player 1. The three values are Red, Green and Blue. Note: range is 0...63d.
003Fh Color player 2 3 * 1 Byte Color settings for player 2. The three values are Red, Green and Blue. Note: range is 0...63d.
0042h Weapon equipment player 1 5 * 1 Byte These values represent the 5 weapons of player 1. Weapons are sorted by weapon index (See Liero offset list). Note: range is 1...40d. Any double weapons will be replaced by a random weapon at the start of the game.
0047h Weapon equipment player 2 5 * 1 Byte These values represent the 5 weapons of player 2. Weapons are sorted by weapon index (See Liero offset list). Note: range is 1...40d. Any double weapons will be replaced by a random weapon at the start of the game.
004Ch Health player 1 1 Short Health setting for player 1.
004Eh Health player 2 1 Short Health setting for player 2.
0050h Length player 1 name 1 Byte Length player 1 name. Liero will only read this number of bytes of the player 1 name data. If this value is 0, Liero will pick a name from NAMES.DAT.
0051h Player 1 name 20 * 1 Byte This is the name of player 1. If the name is less than 20 bytes long, the remaining bytes may be crap data or null bytes.
0065h Length player 2 name 1 Byte Length player 2 name. Liero will only read this number of bytes of the player 2 name data. If this value is 0, Liero will pick a name from NAMES.DAT.
0066h Player 2 name 20 * 1 Byte This is the name of player 2. If the name is less than 20 bytes long, the remaining bytes may be crap data or null bytes.
007Ah Load+Change 1 Byte Load+Change setting. Possible values:
0 = OFF
1 = ON
007Bh Length DAT filename (?) 1 Byte Length DAT filename. Liero will only read this number of bytes of the DAT filename data. (?)
007Ch DAT filename (?) 8 * 1 Byte Liero will internally recognise the DAT file as this name. (?) If the name is less than 8 bytes long, the remaining bytes may be crap data or null bytes.
0084h Player 1 key setup 7 * 1 Byte This is the key setup for player 1. The keys are in this order:
Aim up key
Aim down key
Move left key
Move right key
Fire key
Change key
Jump key
Check the key list (In the useful lists section) to see which keys are available and how they are encoded.
008Bh Player 2 key setup 7 * 1 Byte This is the key setup for player 2. The keys are in this order:
Aim up key
Aim down key
Move left key
Move right key
Fire key
Change key
Jump key
Check the key list (In the useful lists section) to see which keys are available and how they are encoded.
0092h Length level name 1 Byte Length level name. Liero will only read this number of bytes of the level name data.
0093h Level name 8 * 1 Byte This is the level loaded at startup. If the name is less than 8 bytes long, the remaining bytes may be crap data or null bytes.
